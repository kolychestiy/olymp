# Кратчайшие расстояния на графах

Ранее [был рассмотрен](https://github.com/kolychestiy/olymp/tree/main/КБ/1.6%20Поиск%20в%20глубину%20и%20поиск%20в%20ширину#readme) алгоритм поиска в ширину (BFS), позволяющий за линейное время ($\mathcal{O}(n + m)$) искать кратчайшие расстояния в невзвешенном графе от некоторой вершины до всех остальных, где расстояние измерялось в количестве пройденных рёбер.

Настало время поговорить о поиске расстояния в более общем случае, то есть на взвешенном графе. А именно, теперь каждому ребру будет назначен его вес $-$ некоторое число. Длиной пути соответственно будет сумма весов рёбер на пути. Невзвешенный граф является в таком случае частным случаем взвешенного, но у которого всем рёбрам присовен одинаковый вес, равный единице.

И самым важным алгоритмом с точки зрения олимпиадного программирования для поиска расстояний во взвешеных графах является алгоритм Дейкстры. В [этой статье](https://ru.algorithmica.org/cs/shortest-paths/dijkstra/) он достаточно хорошо описан. Используется в большом количестве задач на поиск расстояний от одной вершины до всех остальных во взвешенном графе, и единственным его серъёзным недостатком является то, что он работает быстро и коректно только для графов без рёбер отрицательного веса.

На случай графов с рёбрами отрицательного веса есть другой, более медленный, но работающий уже независимо от весов графа, алгоритм Беллмана. В [этой статье](http://e-maxx.ru/algo/ford_bellman) подробно описан сам алгоритм и его применение для проверки на наличие цикла отрицательного веса или его нахождение.

Наконец, последним предлагаемым к ознакомлению алгоритмом является алгоритм Флойда. В [этой статье](http://e-maxx.ru/algo/floyd_warshall_algorithm) разбирается данный алгоритм, предназначенный для поиска расстояний между всеми парами вершин в графе без циклов отрицательного веса. Данный алгоритм асимптотически не быстрее Дейкстры, но в случае с плотными графами (или просто полными) выигрывает у Дейкстры по константе времени работы. Сравниваня с алгоритмом Форд-Беллмана алгоритм уже выигрывает по времени, но для поиска расстояний от одной вершины до остальных на разреженных графах проигрывает.
